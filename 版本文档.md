# 版本文档

## 版本定位说明

本文件用于记录论文从早期探索稿到当前标杆稿的关键演化轨迹，并明确哪些版本代表了实质性的理论转向。

当前结论：

- `V1` 到 `V7`：探索性迭代版本
- `V8`：第一个真正进入可投稿区间的核心版本
- `标杆V1`：第一版独立 benchmark 版本，完成 `TLRA + image-only` 的正式收敛
- `标杆V2`：上一轮正式 benchmark 版本，在 `标杆V1` 基础上增强 ACM MM 契合表达，并加入次主线 video pilot
- `标杆V3`：在 `标杆V2` 基础上的收口强化版，吸收跨高分窗口稳定成立的协议封口项
- `标杆V4`：在 `标杆V3` 基础上补齐正文图表占位、附录占位、实验状态映射与 4-GPU 调度，现为最新 benchmark
- `V55`：旧自动迭代终点，但已出现明显漂移与 `3/5` 平台
- `V60`：旧的 benchmark reset 起点，锚定 `标杆V2`
- `V75`：新的正式 reviewer/scientist 讨论起点，由 `标杆V3` 直接复制生成

### 旧 `V8_标杆版`

- 文件：`论文大纲_V8_标杆版.md`
- 定位：旧 benchmark 载体文件
- 形成方式：先从 `V8-V10` 的高分稳定窗口中提取结构，再经过一次关键理论纠偏

#### 第一阶段价值

最初的 `V8_标杆版` 价值在于：

- 固化高分阶段的稳定结构
- 压制 `V11+` 的解释债增长
- 保留 `Adaptive Top-k`、`VASM`、公平边界和实验骨架

#### 第二阶段纠偏

用户指出了一个根本性问题：

- 旧标杆稿错误地把 `Pooling Paradox` 推成论文题眼，并错误牵连到 `VCD / DoLa / OPERA`

这不是措辞瑕疵，而是一次理论纠偏。纠偏后的 `V8_标杆版` 已经不再依赖旧题眼，而是重构为一篇正面方法型稿件：

- 主标题不再围绕 `Pooling Paradox`
- 主问题改为：如何在 decode-time 引入 token-local visual evidence，同时保住结构与推理
- 论文主轴收敛为：
  - `BRA` 作为 decode-time / logits-space intervention
  - `Adaptive Top-k` 作为 local evidence extractor
  - `VASM` 作为 structure-preserving protection

## 为什么旧 benchmark 仍然有价值

旧 `V8_标杆版` 仍然重要，不是因为命名合理，而是因为它完成了方向上的更正：

1. 它不再依赖错误描述外部 baseline。
2. 它不再靠一个可能显然或伪命题的问题定义来撑起整篇论文。
3. 它把论文重心移回 BRA 自己真正可验证的精髓。
4. 它把实验逻辑改成“有效性 + 副作用 + 局部证据价值”的三条证据链。

## 新 benchmark 体系

从本轮开始，benchmark 与正式迭代版本彻底分离：

- benchmark 使用：`标杆V1`、`标杆V2`、`标杆V3` ...
- 正式迭代使用：`V38`、`V39`、`V40` ...

对应文件命名规则改为：

- `论文大纲_标杆Vx.md`
- `实验方案_标杆Vx.md`

而不是旧的 `V8_标杆版` 这种把 benchmark 挂在迭代版编号后的写法。

## 对后续版本的约束

后续若继续生成新版本，必须遵守以下约束：

- 不得再把 `Pooling Paradox` 恢复为标题级或摘要级题眼
- 不得再通过错误描述 `VCD / DoLa / OPERA` 来制造论文冲突
- 不得再让无实验支撑的工程实现细节主导正文
- 必须始终把论文核心写成：
  - decode-time
  - token-local
  - structure-preserving

## 当前建议

从现在开始，版本策略应切换为：

- `V8`：保留为首个核心版本
- `标杆V1`：保留为第一版独立 benchmark
- `标杆V2`：保留为重要收敛节点
- `标杆V3`：保留为重要收口版本
- `标杆V4`：上一轮最新 benchmark
- `标杆V5`：上一轮 benchmark，基于 `标杆V4` 的收口式升级版本
- `标杆V6`：当前最新 benchmark，完成从 `Phi_calib` 内耦合叙事到 strict universal sidecar 叙事的理论转向
- `V75`：上一轮官方 latest iteration base
- `V86`：上一轮官方 latest iteration base，由 `标杆V4` 直接复制生成
- `V97`：当前正式 latest iteration base，由 `标杆V5` 直接复制生成
- 后续版本只围绕以下方向推进：
  - 稳定 `TLRA_univ` 作为主方法、`Psi_univ` 作为主学习模块的叙事
  - 将 `TLRA_internal` 明确收束为 control，而不是伪 universality 主角
  - 验证 strict train-once / run-anywhere 是否在无学习型 per-model adapter 的条件下成立
  - 量化 portability / ceiling tradeoff，而不是回避它
  - 用 string-side structural gate 取代 universality 不成立的 vocab-side 主叙事
  - 保留 `Top-M` ceiling、prefix ambiguity、abstention 作为正式失败边界

而不是再次回到旧的宏大题眼叙事。
