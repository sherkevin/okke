# 方法论纠偏文档_V5~V22

## 1. 目的

本文档不是实验进度记录，也不是某一版 `idea/review` 的复述。  
它的作用是从 `veb_idea_V5` 到 `veb_idea_V22` 以及对应 `veb_review` 的往返争论中，抽取对当前 `UniGround / VEB` 路线真正有价值的稳定结论，并据此纠偏当前论文方法论与工程实现。

基本立场如下：

- 审稿人的具体措辞可能偏激；
- 科学家的具体提案可能过满；
- 但两者反复冲突的地方，往往恰恰暴露了最值得保留的高价值原则。

## 2. 从 V5~V22 提炼出的核心共识

### 2.1 最稳定的正确方向不是“内部 counterfactual prior 提取”，而是“外部通用证据约束”

从 `V16` 到 `V22`，大量方案都试图通过如下方式构造“纯文本先验”或“文本惯性”：

- 内部 attention masking
- final-layer visual removal
- intermediate-layer projection
- offline textual subspace extraction
- parallel counterfactual branch

这些方案虽然各自试图修补前一版的问题，但它们共享一个脆弱前提：

> 可以在宿主模型内部，以足够稳定、足够严格、足够可解释的方式隔离出“文本先验”。

对应的 `review` 反复指出：

- 深层多模态特征已经高度纠缠；
- 任何内部结构性扰动都极易引入 OOD 或校准失真；
- “pure textual prior”“zero OOD”“strictly calibrated” 这类表述极易成为 reviewer 攻击点。

因此，对当前路线最重要的纠偏是：

> 不要把方法论的科学核心建立在“我们可以在宿主内部严格提取纯文本先验”上。

当前更稳、更可 defended 的核心应当是：

> 我们通过外部、对象条件化、可审计的视觉证据约束最终答案决策，而不是声称已经在宿主内部完成 prior/evidence 的严格剥离。

### 2.2 `Psi_univ` 的正确身份必须被严格固定

从 `V5` 开始最有价值、也最稳定的结构性思想，是把系统拆成：

- universal evidence scorer
- task adapter
- decision layer

这是整个 `V5~V22` 序列中最应该保留下来的资产。

由此得到的纠偏规则是：

- `Psi_univ` 不是 `POPE` 专用分类器；
- `Psi_univ` 不是宿主耦合 answer head；
- `Psi_univ` 不是 benchmark-specific verifier；
- `Psi_univ` 是通用外部证据评分器。

任务特定行为只能放在：

- object extraction
- hypothesis construction
- answer-step decision layer

而不能被吸收到 `Psi_univ` 本体里。

### 2.3 真正该避免的不是 “Top-K” 本身，而是 “依赖 frontier 暴露目标答案”

`V9/V11/V12/V22` 中关于 sparse projection / Top-K blind spot 的争论虽然服务于另一条技术路线，但其中有一个结论对当前方法非常重要：

> 如果一个方法只有在正确答案已经出现在当前候选池中时才有机会起作用，那么在强 prior 场景下它天然存在失明风险。

对我们当前方法的意义不是：

- 必须做 full-vocab intervention

而是：

- verifier 不应把“yes/no 恰好进入 generic frontier”作为自己生效的前提。

也就是说，当前方法的核心控制对象必须是：

- 明确的 answer interval

而不是：

- 任意 top-k token frontier 中碰巧出现的答案标签。

### 2.4 最该保留的安全机制是“支持不足时不覆盖”，而不是越来越复杂的 penalty 设计

多轮 `idea` 版本不断在设计：

- adaptive contrastive calibration
- exact normalization
- bounded sparse projection
- entropy-scaled penalties
- subspace penalties

这些技术有些局部上有启发，但它们共同的问题是容易把方法的主叙事变成：

> 我们发明了一种更精细的惩罚函数来压制 hallucination。

这不适合当前路线。

当前路线真正应该坚持的是：

> 当外部证据不足时，不去强行改写宿主；当外部证据足够时，只对最终决策做有限且可审计的约束。

因此，`support-aware backoff` 比复杂 penalty 家族更接近当前方法的理论核心。

### 2.5 论文中必须避免的高风险表述已经非常清楚

从 `review` 的攻击模式看，以下表述风险极高，应避免或极度谨慎使用：

- `pure textual prior`
- `strictly isolate`
- `zero OOD`
- `exact calibration`
- `mathematically guaranteed recovery`
- `near-zero latency`
- `complete elimination of hallucination`

更稳妥的替代口径是：

- `external evidence bottleneck`
- `object-conditioned verification`
- `bounded decision control`
- `support-aware backoff`
- `portable external scorer`
- `deterministic task adapter`

## 3. 对当前方法论的纠偏结论

### 3.1 当前主路线总体方向是对的

与 `V16~V22` 的内部 counterfactual 路线相比，当前 `UniGround / VEB` 路线在以下方面更正确：

- 不依赖宿主内部 hidden-state 解释学；
- 不把 universality 建立在内部 attention 操作上；
- 明确把对象验证作为核心任务对象；
- 强调外部视觉证据而非内部 prior 提取；
- 保留了通用模块 `Psi_univ` 的独立边界。

### 3.2 当前主路线仍需纠偏的地方

虽然方向对，但仍有几个必须明确修正的点：

1. 不能把方法重新表述成“generic token-local patching”。
   - 对 `POPE` 来说，核心对象应该是 object-level yes/no verification。

2. 不能让 verifier 的生效依赖 generic frontier 暴露 `yes/no`。
   - 否则会重现 blind-spot 问题。

3. 不能让 `Psi_univ` 的训练目标偷偷退化成 benchmark-specific head。
   - 否则 universality 叙事失效。

4. 不能把论文主张写得过强。
   - 尤其不能暗示已在宿主内部实现 strict prior-evidence separation。

## 4. 对当前代码的直接启示

### 4.1 应保留的实现原则

- `task_adapter` 显式抽取 object
- 构造 `H_yes(o)` / `H_no(o)`
- retrieval 必须 object-conditioned
- intervention 只作用在 answer-step
- intervention 应有 backoff
- 审计必须记录 `host_no_to_yes` / `host_yes_to_no`

### 4.2 应优先修复的实现原则

- verifier 不应依赖 generic top-k frontier 暴露目标答案
- binary control 应该直接对 designated answer labels 作用
- `Psi_univ` 的训练/调用边界必须重新对齐为“通用证据评分器”

## 5. 对论文 Method 章节的纠偏要求

基于以上结论，论文 Method 章节应该满足以下要求：

1. 先讲 universality boundary，再讲任务适配。
2. 明确 `Psi_univ` 是通用证据评分器。
3. 明确 downstream task 只定义：
   - query object
   - hypothesis pair
   - decision rule
4. 明确 answer-step control 是核心，不把 generic frontier patching 当主叙事。
5. 明确 support-aware backoff 是安全机制，而不是后处理附属项。
6. 明确我们不声称：
   - strict internal prior isolation
   - zero OOD
   - exact causal disentanglement

## 6. 最终纠偏后的方法口径

当前最稳妥、最统一、最适合写入论文并指导工程的口径应为：

> 我们的方法不是试图在宿主模型内部提取一个“纯文本先验”并与之对抗，而是通过一个通用外部证据评分器 `Psi_univ`、一个任务特定但非学习的 object adapter，以及一个轻量 answer-step decision layer，在最终答案层面施加对象条件化的视觉证据瓶颈，从而抑制 prior-dominated hallucination。

## 7. 一句话经验教训

从 `V5~V22` 学到的最重要教训不是“哪一种内部 contrastive decoding 更聪明”，而是：

> 越试图在宿主内部严格构造 prior counterfactual，方法越容易变得脆弱、过度宣称、且偏离 universality；越把通用证据评分、任务适配和最终决策边界拆清楚，方法越稳、越容易落地、也越容易 defended。
