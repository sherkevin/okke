## 1. 这版方案为什么成立

`标杆V6` 的核心不是继续打磨旧 `Phi_calib`，而是完成一次更根本的理论收敛：

1. 承认 `Phi_calib` 是 **internal / model-coupled** 路线，不再强写成通用热插拔插件。
2. 将主方法升级为 **`TLRA_univ`**，其唯一学习模块 `Psi_univ` 工作在共享外部语义空间，而不是模型原生 hidden/logit 几何里。
3. 将“train once, run anywhere”从宣传语变成实验合同中的正式可证伪条件。

因此这版实验方案关注的不是数据是否已下载，而是**哪些实验能真正证明 universality claim 成立或失败**。

## 1.1 来自 reviewer / scientist 窗口的六条硬约束

这版 `V6` 不是凭空设计，而是吸收了 `V97-V117` 平台期里跨轮稳定出现的六条洞察。它们必须被视为**正式实验硬约束**：

1. **身份诚实约束**：`TLRA_internal / TLRA_calib / Base + LoRA` 只能作为 control；`TLRA_univ` 才是 universal 主线。
2. **语言克制约束**：实验与文稿中不再过度使用 `routing` 表述，统一改为 grounding control / sidecar verification / decode-time control interface。
3. **`External Global Prior` 约束**：必须加入一个简单但强的全局视觉先验基线，否则无法证明 token-local / candidate-aware 设计的必要性。
4. **`abstention -> abort` 约束**：`abstention` 必须被落实为真正的 back-off 机制，而不是只做诊断输出。
5. **tokenizer bridge 主证据约束**：`prefix ambiguity`、`span collapse`、`suffix stability` 必须进入主资产而不是只放在附录。
6. **latency 拆解约束**：必须拆解 candidate construction / sidecar scoring / bridge redistribution / jitter，而不是只报总吞吐量。

## 2. 当前主论文的四个核心问题

### Q1. `TLRA_univ` 是否真的 train once, run anywhere

这是第一优先级，必须直接回答。

允许变化：

- tokenizer 解析逻辑
- 非训练的字符串 canonicalization
- 非训练的 prefix-to-logit 归因规则

不允许变化：

- `Psi_univ` 权重
- per-model learned adapter
- per-model retraining

### Q2. `TLRA_univ` 是否仍然是 decode-time intervention，而不是后验 reranker

必须证明：

- 插件读取的是生成中的 Top-`M` 候选
- 插件输出的是当步 logits bias
- 最终 token 仍由原 MLLM 解码产生

### Q3. universality 代价是否过大

需要衡量：

- 与 `TLRA_internal_calib` 相比的性能上限差距
- 与 Base / `VCD` / `DoLa` 相比的收益
- 额外时延、显存、候选构造成本

### Q4. 结构保护是否仍然成立

`TLRA_univ` 不再使用 vocab-side VASM，因此必须单独验证：

- string-side structural gate 是否足够稳定
- continuation / suffix 行为是否劣化
- 推理任务是否被误伤

## 3. Required Experimental Axes

### 3.1 Axis A: Module-Level Validation Before System Runs

Goal:

- Verify that `Psi_univ` itself learns reliable `support / contradiction / abstention` judgments in the shared external semantic space.

Rows to compare:

- `Psi_univ checkpoint`
- `HardcodedUniversalScorer`
- `ZeroUniversalScorer`

Mandatory outputs:

- macro-F1
- per-class accuracy
- abstention rate / abstention calibration
- checkpoint contract metadata
- external encoder contract

Interpretation rules:

- If the module is invalid, a later system failure must not be explained away as a runtime-only issue.
- If the module is valid but the system is weak, the problem belongs to the runtime / trigger / bridge / redistribution boundary.

### 3.2 Axis B: System-Level Cross-Model Hot-Swap Results

Goal:

- Reuse the same `Psi_univ` checkpoint across multiple heterogeneous MLLMs.
- Test whether `UniGround` provides net gains without retraining and without learned per-model adapters.

Rows to compare:

- Base
- `VCD`
- `DoLa`
- `TLRA_internal_zero`
- `TLRA_internal_calib`
- `TLRA_univ`

Mandatory outputs:

- hallucination metrics
- intervention coverage
- ITL / TPOT / TTFT
- peak_vram
- shared-checkpoint manifest

Failure rule:

- If any host requires a learned adapter or per-model retraining, the strict universal claim fails.

### 3.3 Axis C: Forced `External Global Prior` Control

Goal:

- Test whether candidate-driven local control is truly stronger than a simple frozen global visual prior.

Rows to compare:

- `TLRA_univ`
- `TLRA_univ_global_only`
- `External_Global_Prior`

Definition constraints:

- `External_Global_Prior` may only use a frozen external encoder to produce an image-level static bias.
- It may not perform candidate-local span scoring.
- It may not access host hidden states.

Interpretation rules:

- If `TLRA_univ` only matches `External_Global_Prior`, the candidate-driven resonance claim must contract.
- If `External_Global_Prior` wins, the local evidence wake-up mechanism needs further redesign.

### 3.4 Axis D: Risk Gate and Structure Protection

Goal:

- Test whether a risk-aware gate is better aligned with real hallucination risk than a pure entropy trigger.
- Test whether the string-side gate replaces the core role of vocab-side VASM.

Rows to compare:

- Base
- `TLRA_univ`
- `TLRA_univ_no_gate`
- `TLRA_univ_no_abstain`
- `TLRA_internal_calib`

Mandatory outputs:

- suffix / continuation stability
- function-word corruption
- OCR-heavy abstention behavior
- general capability retention
- risk-trigger rate
- abort-trigger rate
- abort-after-trigger grammatical survival

Hard rules:

- `abstention` must trigger real intervention abort / back-off behavior.
- A passive `abstention_score` without behavioral change does not count.

### 3.5 Axis E: Evidence Wake-Up Ablations

Goal:

- Determine whether the gain comes from candidate-conditioned evidence wake-up, the risk gate, or only a global image prior.

Mandatory ablations:

- `TLRA_univ_no_gate`
- `TLRA_univ_no_abstain`
- `TLRA_univ_global_only`
- `TLRA_univ_region_only`
- `External_Global_Prior`

Interpretation rules:

- If `global_only` is close to full `TLRA_univ`, the candidate-driven local resonance claim must contract.
- If `region_only` is much weaker, full-image context and local evidence likely need to be modeled jointly.
- If `no_abstain` is stronger, the abstention head needs redesign.

### 3.6 Axis F: Failure Boundaries and Cost Audit

Mandatory audits:

- Top-`M` ceiling
- prefix ambiguity rate
- span collapse errors
- suffix / continuation stability
- latency split:
  - observation caching
  - candidate construction
  - sidecar scoring
  - bridge redistribution
  - jitter / batch sensitivity
- abstention histogram
- abort-trigger rate

This section is not decorative. It is part of the scientific asset behind the universality claim.

## 5. Code and Implementation Contract

### 5.1 Legacy Paths That May Remain

These files still have value, but their identity has changed:

- `train_bra_calib.py`
- `bra_logits_processor.py`
- `bra_operator_multi.py`

They should now be treated as:

- `TLRA_internal` implementation
- control path
- not the carrier of the strict universal claim

### 5.2 The New Path Must Exist Independently

`TLRA_univ` must exist as a real code path rather than a comment-level future idea:

- universal scorer `Psi_univ`
- string-side gate
- parameter-free tokenizer bridge
- risk-aware trigger
- candidate-conditioned evidence wake-up
- decode-time sidecar processor
- explicit abort / back-off logic

### 5.3 v2 Boundary Alignment

The following files must converge to the same method boundary:

- `run_uniground_v2_eval.py`
- `uniground_v2/runtime.py`
- `uniground_v2/trigger.py`
- `uniground_v2/bridge.py`
- `uniground_v2/regions.py`

Alignment principles:

- candidate-conditioned evidence wake-up, not `prefix max-max` differencing
- risk gating, not `high entropy => trigger`
- candidate-level local evidence, not a shared region-pool substitution
- bounded conservative redistribution, not an all-purpose correction narrative

### 5.4 Strictly Forbidden

`TLRA_univ` must not:

- consume model-native hidden states as learned input
- depend on `lm_head.weight`
- train a separate learned adapter for each host model
- describe host internal attention steering as if it were already the implemented main method

## 6. 结果解释规则

### 如果 `TLRA_univ` 成功

可以主张：

- universal train-once plugin 是可行的
- universality 需要 external semantic space，而不是 hidden/logit alignment
- candidate-aware control 明显优于简单 global prior

### 如果 `TLRA_univ` 中等成功

可以主张：

- strict universal route 成立，但 ceiling 低于 internal route
- 论文核心价值转为 portability / ceiling tradeoff 与新设计原则
- 或者 candidate-aware 仅在部分场景优于 global prior，需要收缩主张

### 如果 `TLRA_univ` 失败

也不能退回旧的伪 universality 叙述。只能诚实写成：

- `TLRA_internal` 有效，但 strict universal route 仍未闭合
- 失败边界来自 prefix ambiguity / external encoder blind spots / Top-`M` ceiling / global-prior indistinguishability

## 7. 当前正式判断

从 `标杆V6` 开始，论文主线应明确切换为：

- **主方法：`TLRA_univ`**
- **主学习模块：`Psi_univ`**
- **旧方法角色：`TLRA_internal` control**

这比继续强写 `Phi_calib` 的通用性更厚、更新，也更诚实。
