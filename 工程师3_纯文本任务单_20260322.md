# 工程师3纯文本任务单

你现在的任务不是跑更多实验，而是把 `标杆V3` 已经写进实验合同、但当前代码还没有真正闭环的部分补上。请严格按照下面优先级执行。除非我后续明确变更，否则你默认**不占用主实验 GPU 长跑资源**；你的主要工作是代码实现、轻量 smoke、结果结构补齐。

## 任务原则

1. 你的目标是让 `标杆V3` 的关键实验合同变成“真实可执行、可审计、可导表”的代码，而不是发明新叙事。
2. 你优先修 `P0`，只在 `P0` 收口后再做 `P1`。
3. 你默认不抢占工程师1和工程师2的主长跑 GPU。
4. 只有在验证 loader / 方法入口 / smoke test 时才使用 GPU。

## GPU 使用规则

- `GPU0`：保留给 reviewer/scientist 工作流或主线控制面，不要占用。
- `GPU1`：保留给 `Chain A` 长跑，不要占用。
- `GPU2`：保留给 `Chain B / FREAK` 正式实验，不要长期占用。
- `GPU3`：仅允许你做短时 smoke / loader 验证 / 方法入口验证。
- 你的默认工作模式：**CPU + 本地代码修改**。
