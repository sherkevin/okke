# Token-Local Resonance Anchoring: Structure-Preserving Decode-Time Intervention for MLLM Grounding

## Abstract
Multimodal Large Language Models (MLLMs) frequently hallucinate when strong language priors overpower the localized visual evidence required for grounding physical entities. Existing inference-time mitigation strategies—such as contrastive decoding, attention control, and layer-contrast methods—are highly competitive sequence-level heuristics. However, they leave open the question of whether injecting explicit *token-local visual evidence* directly into decode-time logits can improve physical object grounding without collapsing linguistic structure. We present **Token-Local Resonance Anchoring (TLRA)**, a decode-time intervention protocol that reweights candidate tokens using localized spatial support extracted from contextualized visual states.

TLRA operates on three structural commitments. First, it extracts token-specific spatial support via an **Adaptive Top-$k$** mechanism over a strictly bounded candidate window ($M$). Second, to avoid the prohibitive latency of autoregressive part-of-speech tagging, we introduce **Vocabulary-Anchored Semantic Masking (VASM)**—an offline, pre-computed WordNet static mask coupled with dynamic $O(1)$ Byte-Pair Encoding (BPE) continuation inheritance, ensuring functional syntax and abstract reasoning tokens are mathematically bypassed. Third, acknowledging that native visual and lexical states lack perfect alignment, we structurally separate our method into `TLRA_zero` (a pure zero-shot viability probe) and `TLRA_calib` (a Test-Time Adaptation module using a neutrally-trained projection). To ensure rigorous scientific claims, we enforce a strict evaluation protocol: auditing Average Generation Length (AGL) to reject truncation artifacts, utilizing DocVQA as a mathematical negative control, replacing numerically unstable logit-range penalties with robust dispersion scaling, and testing spatial routing against a calibrated MeanPool baseline on strictly unseen object categories.

## 1. Introduction

Multimodal Large Language Models (MLLMs) exhibit remarkable reasoning capabilities, yet their autoregressive decoding remains vulnerable to hallucination. The model frequently generates text that is fluent and plausible but weakly supported by the actual visual input. This failure is acutely problematic in the grounding of physical entities—such as small-object recognition, spatial positioning, and counting—where the necessary visual evidence is highly localized in the patch space.

Current inference-time mitigation strategies represent major advances in the field and serve as our primary competitive baselines. Contrastive decoding methods like VCD contrast different input noise conditions to reshape distributions; attention-based control methods like OPERA penalize abnormal generation patterns; and layer-contrast methods like DoLa exploit differences between shallow and deep representations. These are highly effective *sequence-level or distribution-level language heuristics*. However, they leave a specific, complementary proposition untested: 

**Can we inject token-local spatial visual evidence into decode-time logits to outperform sequence-level heuristics on physical grounding, without destroying language structure or incurring catastrophic latency?**

Executing this token-local intervention is mechanically non-trivial due to three explicit bottlenecks:
1. **The Autoregressive Latency Paradox:** Running NLP pipelines (e.g., POS taggers) to isolate physical entity tokens at decode-time is fundamentally flawed. Taggers require complete sentences to resolve ambiguity, and executing them dynamically for $M$ candidate tokens per step incurs catastrophic latency.
2. **Embedding Asymmetry:** Visual patch states and output logits do not natively align. Bridging this gap purely zero-shot is often noisy, necessitating a structural split between pure zero-shot probes and externally calibrated adaptation modules.
3. **Candidate Bound & Memory Bandwidth:** Post-hoc intervention can only reweight tokens the base model keeps alive in its top-$M$ predictions. Dynamically retrieving spatial support for $M$ candidates introduces a severe memory-bandwidth (Time-Per-Output-Token, TPOT) tax.

To address these challenges, we introduce **Token-Local Resonance Anchoring (TLRA)**. TLRA utilizes an **Adaptive Top-$k$** mechanism to dynamically route localized patch support to candidate tokens within a bounded logit window. To solve the latency paradox, we introduce **Vocabulary-Anchored Semantic Masking (VASM)**—an offline, pre-computed static dictionary mapping strict WordNet hypernym paths, combined with an $O(1)$ BPE inheritance state machine. VASM mathematically zeroes out intervention on syntax, verbs, and abstract concepts, protecting the model's structural integrity.

To ensure methodological rigor, we explicitly divide our approach: `TLRA_zero` acts as a pure zero-shot viability probe to compare against training-free baselines, while `TLRA_calib` (utilizing a neutrally trained feature translator) operates as a Test-Time Adaptation (TTA) module. We commit to a strict, hypothesis-driven evaluation protocol built on three evidence chains: validating hallucination reduction via an Average Generation Length (AGL) audit; confirming structure preservation by using dense text OCR (`DocVQA`) purely as a flat-line negative control; and proving the efficacy of *local* evidence by demanding our spatial routing outperform a calibrated global pool (`MeanPool`) strictly on *unseen* object categories.

## 2. Related Work

### 2.1 Decode-Time Hallucination Mitigation

Inference-time mitigation for MLLM hallucination has developed along several lines. Contrastive methods such as `VCD` compare logits produced under different visual conditions. Attention-based methods such as `OPERA` detect and penalize pathological generation behavior during the autoregressive process. Layer-contrast methods such as `DoLa` compare distributions from different network depths. These methods demonstrate that hallucination can be mitigated at inference time, but they rely on different intervention signals and involve different latency, robustness, and implementation trade-offs.

### 2.2 Local Evidence vs. Coarse Control Signals

High-resolution MLLMs no longer operate over a single compact visual descriptor. They operate over large spatial token fields, and in video settings they further inherit spatio-temporal structure. For dense grounding tasks, the useful evidence for a word is often concentrated in a small region of the image or a short temporal window. This makes token-local evidence attractive: if the next token is visually supported, the support is often sparse, local, and candidate-dependent. TLRA is built around the claim that decode-time hallucination mitigation should take that locality seriously.

This is not a sweeping claim about all existing baselines. It is a narrower methodological claim: when the task depends on local visual evidence, a framework that explicitly reweights candidate logits using localized support is worth testing against coarser internal alternatives. That claim is experimentally verifiable through internal ablations such as `TLRA_zero` vs. `TLRA_MeanPool`.

### 2.3 Structure Preservation in Decoding

A recurring difficulty in token-level intervention is that not all tokens should be treated equally. Functional syntax, discourse markers, and continuation fragments play a structural role that differs from semantically grounded object words or OCR-bearing entities. A practical decode-time method must therefore distinguish hallucination suppression from indiscriminate token suppression. TLRA addresses this through a deterministic structural mask rather than entropy-only heuristics.

## 3. Methodology

### 3.1 Fairness Boundary: `TLRA_zero` vs. `TLRA_calib`

Because visual patch states and output logits often lack native alignment, we separate TLRA into two tracks to prevent "calibrator blackbox" ambiguity:

1. **`TLRA_zero` (Zero-Shot Probe)**: The base model's native language head is applied directly to contextualized visual states. This track serves primarily as a parameter-isolated viability probe and is the only variant directly comparable to pure zero-shot heuristics (DoLa, VCD).
2. **`TLRA_calib` (Lightweight TTA / PEFT)**: The operational variant utilizing a 2-layer MLP projection, $\Phi_{calib}$, to bridge embedding asymmetry. Crucially, to ensure a fair spatial parity test, $\Phi_{calib}$ is trained strictly via standard next-token prediction (cross-entropy) on 50k conceptual captions *without* the TLRA decode-time logit intervention active. It acts solely as a neutral feature translator. We explicitly classify `TLRA_calib` as a Test-Time Adaptation module. Its primary ablation baseline is `TLRA_MeanPool`—a variant using the exact same $\Phi_{calib}$ weights but globally pooled visual features.

### 3.2 Bounded Candidate Filtering and Adaptive Top-$k$ Resonance

At decoding step $t$, TLRA restricts its intervention to a bounded candidate set: $\mathcal{C}_t = \text{Top-}M(L_{orig})$, where $M \ll |V|$ is a strictly locked candidate window (e.g., $M=50$). 

Let $h_L^{(v_j)}$ denote the contextualized final-layer visual states, with visual token volume $N_v = H \times W$. For each candidate token $c \in \mathcal{C}_t$, we first compute the alignment score with each spatial patch $v_j$. We define $\text{score}(v_j, c)$ as the cosine similarity between their respective representations:
$$
\text{score}(v_j, c) = \frac{\phi_v(v_j) \cdot \phi_t(c)}{\|\phi_v(v_j)\| \|\phi_t(c)\|}
$$
In `TLRA_zero`, $\phi_v$ and $\phi_t$ rely on the base model's native visual projection and word embeddings. In `TLRA_calib`, both are mapped into a shared semantic space via $\Phi_{calib}$.

TLRA then computes local support via an Adaptive Top-$k$ rule, determined by a locked spatial ratio $\rho$:
$$
k = \max\Big(k_{min}, \lceil \rho \cdot N_v \rceil\Big)
$$
$$
S_{raw}(c) = \frac{1}{k}\sum_{j \in \text{Top-}k}\frac{\text{score}(v_j, c)}{\tau_{sim}}
$$

### 3.3 Structure Preservation via Offline VASM and BPE Inheritance

Naive token-level intervention suppresses critical syntax. To avoid the severe latency of decode-time NLP tagging, TLRA introduces **Vocabulary-Anchored Semantic Masking (VASM)**. 

**1. Offline Static Dictionary:** Prior to inference, the entire vocabulary ($|V| \approx 32k - 128k$) is cross-referenced against WordNet. We employ a strict programmatic definition: only tokens whose primary synsets trace an unbroken hypernym path to `physical_entity.n.01` or `object.n.01` are assigned a mask value $\gamma = 1$. Function words, verbs, and abstract syntax are permanently assigned $\gamma = 0$. 

*Limitation on Polysemy:* We explicitly acknowledge that VASM evaluates only the *primary* synset. Polysemous words (e.g., "monitor" or "bat") may be misclassified depending on their dominant usage. This lexical rigidity is a deliberate engineering tradeoff, prioritizing $O(1)$ latency over perfect contextual disambiguation.

**2. Dynamic BPE Continuation Inheritance:** Modern tokenizers fragment words. To handle subwords at decode step $t$ for candidate $c_t$, VASM applies an $O(1)$ inheritance rule:
$$
\gamma(c_t) = \begin{cases} 
\gamma(c_{t-1}^{\text{root}}), & \text{if } c_t \text{ is a continuation subword (e.g., lacks space prefix or has '##')} \\
\text{Lookup}(c_t \text{ in WordNet}), & \text{otherwise}
\end{cases}
$$
Hyphenated words or severe BPE fragmentations that fail exact WordNet matches safely default to $\gamma=0$, intentionally missing an intervention rather than corrupting syntax.

### 3.4 Stabilized Logit Intervention

TLRA converts raw local support into a relative evidence score over the candidate set:
$$
\hat S(c) = \text{Softmax}\big(S_{raw}(c) / \tau_{evidence} \mid c \in \mathcal{C}_t\big)
$$
To apply the penalty, prior formulations relying on the absolute logit range ($\max - \min$) proved numerically unstable, as a single improbable token in the Top-$M$ could massively inflate the intervention magnitude. Instead, TLRA dynamically scales the penalty using the standard deviation of the candidate logits, $\sigma_L = \text{Std}(L_{orig}(\mathcal{C}_t))$. The final logit is explicitly gated by the VASM mask $\gamma(c)$:
$$
L_{final}(c) = L_{orig}(c) - \alpha \cdot \gamma(c) \cdot \sigma_L \cdot (1 - \hat S(c))
$$
If a token is functional syntax ($\gamma(c)=0$), the penalty zeroes out entirely, yielding $L_{final}(c) = L_{orig}(c)$, protecting linguistic structure mathematically.

## 4. Evaluation Protocol and Experimental Design

To ensure methodological transparency, our evaluation is structured as a rigorous protocol mapping explicit hypotheses to specific evidence chains. All hyperparameters ($\tau_{sim}, \tau_{evidence}, \alpha, M, \rho$) are strictly locked uniformly across all tasks based on a single validation set.

### 4.1 Evidence Chain A: Hallucination Reduction and Truncation Audit
**Hypothesis:** Token-local evidence reduces physical hallucination more effectively than sequence-level heuristics, without artificially inflating precision through premature truncation.
- **Protocol (Table 1 Planned):** Evaluated on POPE (Accuracy, F1) and CHAIR (CHAIRi, CHAIRs). 
  - *Zero-Shot Group:* Base, VCD, OPERA, DoLa, and `TLRA_zero`.
  - *TTA Group:* `TLRA_MeanPool`, `TLRA_AdaptiveTopK`.
- **The AGL Audit:** We introduce a mandatory column for **Average Generation Length (AGL)** measured in words/tokens, placed immediately adjacent to the F1 scores. 
- **Validation Criteria:** `TLRA_AdaptiveTopK` must dominate the TTA group. Furthermore, if any TLRA variant reduces AGL by >10% compared to the Base model, the hallucination reduction claim must be downgraded, indicating a "silence engine" truncation artifact rather than a true grounding improvement.

### 4.2 Evidence Chain B: Structure Preservation and Negative Controls
**Hypothesis:** VASM mathematically protects abstract reasoning, and its deliberate blindness to dense text prevents unintended corruption of non-spatial tasks.
- **Protocol (Table 2 Planned):** Evaluated on general multimodal benchmarks: MMBench V1.1, MME, MMMU (Hard), and explicitly **DocVQA**. We ablate Base, `TLRA_no_VASM`, and `TLRA_full` (with offline VASM).
- **Validation Criteria:** 
  1. `TLRA_no_VASM` must show a catastrophic drop on reasoning tasks (MMMU), proving VASM is computationally necessary.
  2. `DocVQA` must remain strictly *flat* ($\Delta \approx 0$) between Base and `TLRA_full`. If `TLRA_full` drops, VASM is erroneously penalizing OCR tokens. Any claim of "improving OCR" is explicitly prohibited by the protocol boundaries.

### 4.3 Evidence Chain C: Local Evidence Value (The Parity Test)
**Hypothesis:** Spatial locality (Adaptive Top-$k$) directly drives grounding performance, independent of global calibration priors or parameter injection.
- **Protocol (Table 3 Planned):** Evaluated on FREAK (Spatial Relations, Object Existence, Counting). The table is explicitly divided into two macro-columns: **"Seen Categories"** (objects present in $\Phi_{calib}$'s dataset) and **"Unseen Categories"**. We benchmark `TLRA_MeanPool` against `TLRA_AdaptiveTopK`, forcing both to use the exact same frozen, neutrally-trained $\Phi_{calib}$ weights, and report the $\Delta$ (`AdaptiveTopK` - `MeanPool`).
- **Validation Criteria (Contingency Plan):** The core scientific claim derives solely from this parity test. `AdaptiveTopK` must statistically outperform `MeanPool` specifically on the *Unseen* split. If it only wins on the "Seen" split, the spatial routing hypothesis is invalidated as a data-leakage artifact. In such an event, the paper's primary claim will gracefully devolve to establishing `MeanPool + VASM` as a safe, fast, and structure-preserving TTA.

### 4.4 Systemic Latency Pareto and Failure Mode Audit
To quantify the memory-bandwidth tax of extracting spatial support for $M$ candidates per step:
- **Systemic Cost Pareto (Figure 2 Planned):** A scatter plot mapping Time-Per-Output-Token (TPOT in ms on an A100 GPU, X-axis, **Batch Size = 1**) against POPE F1 (Y-axis), plotting Base, VCD, DoLa, and TLRA.
- **Candidate Window Ablation (Table 4 Planned):** A dedicated ablation varying the candidate window $M \in \{10, 20, 50, 100\}$ to empirically justify the TPOT vs. Out-of-Candidate Failure tradeoff.
- **Out-of-Candidate Failure Mode (Figure 3 Planned):** A line graph tracking the raw logit rank of a correct physical entity token across generation steps for a failure case. This visualizes the precise decoding step where a target token drops below rank $M$, rendering it mathematically unrecoverable by post-hoc intervention.

## 5. Conclusion & Limitations

The Token-Local Resonance Anchoring (TLRA) protocol offers a highly structured, falsifiable approach to decode-time intervention in MLLMs. By architecting Vocabulary-Anchored Semantic Masking (VASM) as an offline static dictionary tracking `physical_entity.n.01` hypernym paths with $O(1)$ BPE inheritance, we resolve the autoregressive latency paradox that plagues NLP-dependent controls. Furthermore, by rigorously separating our evaluations into a zero-shot probe (`TLRA_zero`) and a neutrally-trained TTA module (`TLRA_calib`), and committing to a calibrated MeanPool parity test on strictly unseen object categories, we isolate the true value of spatial routing from mere parameter calibration.

The integrity of this framework is governed by several explicit limitations:
1. **Calibrator Category Leakage and Contingency:** Because $\Phi_{calib}$ leverages 50k captions, TLRA relies heavily on the strictly partitioned Seen vs. Unseen parity test (Table 3). If local routing fails to generalize to unseen objects, the core claim must structurally devolve from "superior spatial routing" to "structure-preserving test-time adaptation."
2. **Lexical Rigidity and Polysemy:** The static nature of VASM guarantees speed but introduces lexical rigidity. Relying strictly on primary WordNet synsets means polysemous words (e.g., "bat") may be misclassified. VASM prioritizes latency over perfect contextual disambiguation.
3. **Out-of-Candidate Bounds:** Decode-time reweighting mathematically cannot recover an entity token if the base model has already discarded it from the initial Top-$M$ candidate pool (audited in Figure 3).
4. **Systemic Latency (TPOT Tax):** Dynamically retrieving spatial support for $M$ candidates per token imposes a memory-bandwidth bottleneck, translating to an unavoidable TPOT overhead relative to the base model, particularly noticeable at batch size 1.
5. **Modality Extensions:** The core mechanism is optimized strictly for static spatial ($H \times W$) reasoning. Spatio-temporal extensions (e.g., video) lack native token alignment and remain an exploratory secondary pilot (detailed in Appendix) rather than a primary capability.

Ultimately, this protocol establishes a transparent blueprint: proving that token-local visual evidence can reduce hallucination without destroying language structure, provided it is governed by rigorous bounds, strict negative controls, and mathematically sound syntax protection.
