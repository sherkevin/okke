# Breaking the Language Inertia: Mitigating MLLM Hallucinations via Bi-directional Resonance Anchoring

## Abstract

多模态大语言模型（MLLMs）在图像描述、视觉问答、网页理解与视频推理中仍会频繁生成缺乏视觉证据支撑的内容。现有推理期干预方法通常面临两类核心约束：一类依赖双路前向或回溯式搜索，带来显著的时延与算力膨胀；另一类在隐藏状态空间进行统一方向的减法或投影，往往需要对高分辨率视觉 token 做聚合，从而破坏细粒度空间信息。基于此，我们保留 BRA（Bi-directional Resonance Anchoring）在 logits 空间做轻量干预的基本思想，并进一步提出一个更稳健的版本：BRA-V2。

BRA-V2 的核心改进有三点。第一，我们将原始的单 token 视觉共振分数扩展为**相对共振重塑（relative resonance reshaping）**：候选 token 的惩罚与奖励不再只取决于其绝对共振值，而取决于其在当前 Top-K 候选集中的相对证据位置，从而降低统一缩放带来的误伤风险。第二，我们引入**BPE-aware token damping**，对连续子词片段、功能词、标点词与抽象逻辑词采用更保守的惩罚系数，缓解密集 OCR 与长实体生成中的碎片化问题。第三，在视频分支中，我们使用**camera-motion-suppressed temporal difference** 替代朴素的逐帧绝对位置差分，通过去除帧间公共运动分量来提升动作证据的信噪比。

在实验设计上，V2 不再预设“全面 SOTA”这一结论，而是围绕三个更可检验的命题展开：BRA 是否能在 POPE/CHAIR/FREAK 上提高视觉忠实度；BPE-aware damping 是否能降低 OCR/实体碎片误伤；时序差分增强是否能在 VidHalluc / MVBench 中改善动作相关错误。当前原型结果表明，BRA 的收益具有任务依赖性，尚不足以支撑强结论，但其“零池化、低额外开销、可插拔”的工程形态使其仍然有望成为面向高分辨率 MLLM 的底层安全校准模块。

## 1. Introduction

多模态大语言模型的一个长期难题，是**语言先验与视觉证据之间的竞争失衡**。在自回归生成中，模型既要维持语言流畅性，又要忠实于视觉输入。当语言先验在局部时间步压倒视觉证据时，模型就会生成“看起来合理、实际上不在图像或视频里”的内容，即多模态幻觉。

现有缓解路线主要有两类。第一类是基于对比、回溯或多路采样的推理期搜索方法，例如 VCD、OPERA、DoLa 及其衍生变体。它们能在一定程度上减少幻觉，但往往依赖额外前向传播、注意力监控或回滚机制，难以满足实时系统的时延预算。第二类是基于表征工程或子空间干预的隐藏状态修正方法。它们在纯文本领域颇具启发性，但在高分辨率 MLLM 上常常需要对视觉 token 做池化或压缩，从而引发我们强调的 pooling paradox：为了施加统一的向量干预，先摧毁了最需要保留的局部坐标信息。

BRA 的原始出发点是合理的：不要在隐藏状态里做粗暴减法，而是在更靠近输出的 logits / unembedding 空间，通过候选词与视觉特征之间的共振关系，直接校正下一个 token 的排序。然而，第一版 BRA 仍存在三个关键缺陷。其一，它把“单 token 等价于完整语义单元”作为默认前提，难以处理 BPE 碎片与 OCR 长实体。其二，它将 entropy jump 直接视为 hallucination 风险信号，触发依据过于单一。其三，它在视频场景中使用朴素的帧差分，对相机运动与背景平移缺乏鲁棒性。

因此，本文的目标不是推翻 BRA，而是在保持其核心范式的前提下完成一次保守而实质的升级。我们提出 BRA-V2：一个仍然保持 zero-pooling、training-free、decode-time 形态，但在 token 粒度、触发机制和视频时序建模上更稳健的版本。

本文的贡献有三点：

1. 我们将 BRA 的干预单位从“绝对 token 共振”升级为“**相对证据校准**”，使其更像一个局部决策校正器，而不是统一比例缩放器。
2. 我们提出**BPE-aware damping**，在不引入额外训练和复杂 look-ahead 搜索的前提下，对子词连续体、功能词和抽象逻辑词实行保守保护。
3. 我们将视频分支拓展为**camera-motion-suppressed temporal resonance**，在不引入重型时空模块的前提下改善动作类 token 的证据提取。

## 2. Related Work

### 2.1 Inference-Time Intervention for MLLMs

针对 MLLM 幻觉的推理期方法大致可分为三类。第一类是对比解码，如 VCD 通过干净图像和噪声图像两路前向的 logits 差异进行校正。第二类是注意力级回溯方法，如 OPERA 通过监控注意力聚集模式，对过度信任的生成路径施加惩罚。第三类是层间对比方法，如 DoLa 通过成熟层与未成熟层的分布差异来进行对数空间干预。

这些方法的共同问题在于：它们往往以更高的推理代价换取更保守的输出。在高吞吐或实时系统中，这种代价很容易超过可部署边界。对于网页代理、GUI grounding 或长视频推理场景，几十毫秒到数秒级的额外延迟会直接破坏 OODA 闭环。

### 2.2 Representation Engineering and the Pooling Paradox

与解码搜索不同，隐藏状态干预强调从表征几何中找到“真实性方向”或“偏见方向”，并在推理期做最小幅度修正。这类方法在纯文本上富有启发性，但在高分辨率 MLLM 上遭遇新的结构性问题：视觉分支不再是一个紧凑的全局向量，而是保留大规模空间排列的 token 网格。如果为了做单向量干预而先进行均值池化或 token 压缩，那么最需要保留的局部位置与细粒度 OCR 线索就会在干预前被破坏。

我们将这种问题称为 pooling paradox：**为了修正 hallucination，先破坏 grounding。** BRA 的“zero-pooling logits-side intervention”正是对这一悖论的回应。

### 2.3 Temporal Hallucination in Video MLLMs

视频场景中的幻觉不只是“看见了不存在的物体”，还包括“看见了不存在的动作”。近年的工作已经开始强调动作-场景混淆、时间顺序错误以及长程事件因果失配。一个常见但危险的简化，是将时间维度扁平化为静态特征平均，或将相邻帧在绝对坐标上直接相减。这样的设计在存在 camera motion、zoom、pan 或主体位移时，很容易把背景变化误当成动作证据。

因此，视频版 BRA 的目标不应是简单叠加一条 temporal branch，而应是构造一个尽量保守、尽量抑制公共运动噪声的 temporal resonance。

## 3. Methodology

### 3.1 Core Principle

BRA-V2 保留原始 BRA 的核心信念：在 logits 空间而非隐藏状态空间进行干预，从而避免统一的隐空间减法及其带来的语言退化与空间信息损失。给定当前生成步的候选集，我们只处理 Top-K 个候选 token，使计算保持轻量。

设当前步的候选集合为 $\mathcal{C}_t = \{c_1, \dots, c_K\}$，其原始 logits 为 $L_{orig}(c)$。对每个候选 token，我们首先计算其与完整视觉特征集合 $Z_{vision} = \{z_j\}_{j=1}^{N_v}$ 的最大余弦共振分数：

$$
S_{raw}(c) = \max_j \cos(w_c, \bar z_j)
$$

其中 $w_c$ 为 token 对应的 unembedding 向量，$\bar z_j$ 为归一化后的深层视觉特征。

### 3.2 Relative Resonance Reshaping

BRA-V1 的问题在于，单独使用 $S_{raw}(c)$ 会把“低共振”直接解释为“应被惩罚”，这会在功能词、连续子词片段和抽象逻辑词上制造误伤。V2 因此改用**相对共振**：

$$
\hat S(c) = \text{Norm}_{\mathcal{C}_t}\big(S_{raw}(c)\big)
$$

其中 $\text{Norm}_{\mathcal{C}_t}$ 表示在当前 Top-K 候选集内部做稳健归一化。它并不试图给出“这个 token 是否有绝对视觉对应物”的终局判断，而是回答一个更局部的问题：**在当前竞争的几个候选词里，谁更像是被视觉证据支持的那个。**

基于此，我们将 logits 重塑写成：

$$
L_{final}(c)=L_{orig}(c)-\alpha \cdot \gamma(c)\cdot (1-\hat S(c))\cdot |L_{orig}(c)|
+\beta \cdot \eta(c)\cdot \hat S(c)\cdot |L_{orig}(c)|
$$

其中 $\gamma(c)$ 和 $\eta(c)$ 是 token-aware 的保守系数。和 V1 相比，这一版本的惩罚与奖励都不再完全依赖绝对共振，而是依赖相对证据位置与 token 类型。

### 3.3 BPE-aware Token Damping

为了缓解 BPE 碎片化问题，我们不再把所有 token 都视作“应当被像实体词一样校正”的对象。我们将候选 token 划分为四类：

1. **视觉实体/动作词**：允许正常使用相对共振做奖惩。
2. **连续子词片段（continuation subwords）**：如无词边界的 BPE 续写片段，对其使用显著更低的惩罚系数。
3. **功能词、标点和高频连接词**：默认采用保守惩罚，避免破坏句法骨架。
4. **抽象逻辑词**：如 `therefore`, `however`, `contradicts` 等，不要求其必须拥有强视觉对应物，而是使用小惩罚或近似冻结。

这一设计并没有完全解决“多 token 实体的整体共振”问题，但它把原先最危险的“碎片一出来就被打死”改为“碎片被保守对待，给后续 continuation 留出空间”。这是一种工程上更稳定、也更符合当前实现约束的 V2 方案。

### 3.4 Triggering: From Entropy-Only to Uncertainty + Evidence Deficit

V1 把 entropy jump 作为唯一触发条件，这过于脆弱。V2 使用三个触发信号的并联：

1. **Entropy delta**：$\Delta E_t > \epsilon$，表示分布变化较大。
2. **Top-1 margin is small**：当前前两名候选 logits 间距过小，说明模型本身在犹豫。
3. **Evidence deficit**：当前 top-1 候选的共振明显低于其他竞争候选，说明语言先验可能正在压制视觉证据。

只有当这三类信号中的至少一类成立时，BRA-V2 才会进行重塑。这样可以减少“正常语义转折被误触发”的情况。

### 3.5 Video Extension: Camera-Motion-Suppressed Temporal Resonance

为保持 BRA 的轻量特性，我们仍然使用差分式时序特征，但对其进行更保守的处理。对每个视频片段的空间对齐网格，我们计算：

$$
\Delta z^{(t)} = z^{(t)} - z^{(t-1)}
$$

随后减去帧级公共运动分量：

$$
\tilde z_\Delta^{(t)} = \Delta z^{(t)} - \text{Mean}_{spatial}(\Delta z^{(t)})
$$

并可选地加入 stride-2 的多尺度差分以提升对较慢动作的感知。最终视频共振写为：

$$
S_{video}(c)=\lambda_1 \max_i \cos(w_c, z_{vision}^{(i)}) + \lambda_2 \max_m \cos(w_c, \tilde z_\Delta^{(m)})
$$

该分支并不宣称解决所有长程时序因果问题，但它相较于原始的一阶绝对位置差分，更符合开放世界视频中的实际噪声结构。

## 4. Experimental Design

### 4.1 Experimental Philosophy

V2 的实验目标不再是直接宣称全面 SOTA，而是验证以下三个可检验命题：

1. **有效性**：BRA-V2 是否能在 POPE / CHAIR / FREAK 等 grounding-sensitive 任务上减少语言先验主导的错误。
2. **稳定性**：BRA-V2 是否能在 MMBench / MME / MMMU 等通用能力与抽象推理任务上保持小扰动。
3. **可部署性**：BRA-V2 是否能在较低额外时延和有限显存增量下运行。

### 4.2 Models and Baselines

主力模型选择 Qwen3-VL-2B / 8B 与 LLaVA-1.5-7B。对于图像任务，对比基线包括：

- Base decoding
- VCD
- OPERA
- DoLa
- BRA-V1
- BRA-V2

若 InstructBLIP 与 MiniGPT-4 依赖未完整，则在当前阶段只作为扩展验证对象，不作为核心主表主结论来源。

### 4.3 Benchmark Matrix

我们将实验分为三层：

1. **Grounding-sensitive image benchmarks**
   - POPE
   - CHAIR
   - FREAK
2. **General capability preservation**
   - MMBench
   - MME
   - MMMU (hard subset / pilot subset)
3. **Temporal and video grounding**
   - VidHalluc
   - MVBench

### 4.4 Key Ablations

V2 至少需要四组关键消融，而不能只报“BRA 开/关”：

1. `BRA-V1` vs `BRA-V2(relative trigger only)`
2. `BRA-V2` vs `BRA-V2 + BPE-aware damping`
3. `BRA-V2 video(static only)` vs `BRA-V2 video(static + temporal)`
4. `BRA-V2 temporal(raw diff)` vs `BRA-V2 temporal(centered / multi-scale diff)`

### 4.5 Metrics

除标准任务指标外，V2 额外报告：

- `AGL`：平均生成长度，检验是否出现长度塌缩
- `ITL ms/token`：单 token 平均推理时间
- `Peak VRAM`
- `Trigger rate`：BRA-V2 在生成中实际触发的比例
- `Continuation-token damp ratio`：BPE-aware 保护机制作用频率

### 4.6 Current Prototype Evidence and Development Use

当前已有的 N=50 级别 pilot 结果只能作为开发证据，不能直接当成论文主结论。现阶段从现有实验可以得出三点保守判断：

1. BRA 并非在所有模型与数据集上稳定增益，收益具有明显任务依赖性。
2. 当前版本在 MMBench/MME 上通常不会造成灾难性退化，但在 POPE/MMMU 上存在回退风险。
3. 评测协议与实现路径必须统一到同一条 BRA 集成方式，否则结果不可比。

因此，V2 的论文写作应把这些 pilot 结果定位为**development evidence**，而不是 final claim。

### 4.7 Targeted New Experiments for V2

为了支撑 V2 的新方法，新增三类实验最关键：

1. **BPE/OCR 定向实验**
   - 在 FREAK OCR / Position 子集上统计 BRA-V1 与 V2 的错误类型。
   - 重点观察长实体、连写词、数字串是否减少碎片化失败。

2. **Trigger audit**
   - 统计 entropy-only 与 relative-trigger 在 POPE / MMMU 上的触发率、触发位置和误触发案例。
   - 证明 V2 的触发更集中在“模型不确定且证据不足”的时刻。

3. **Video motion robustness**
   - 在 VidHalluc 或 MVBench 动作子任务中，对比 raw temporal diff 与 centered temporal diff。
   - 给出相机运动明显的案例分析，而不是只报 overall score。

## 5. Discussion

BRA-V2 的价值不在于“替代所有现有防幻觉方法”，而在于定义了一种更适合高分辨率 MLLM 的轻量级干预形态：它不需要双路前向，不依赖离线探针训练，也不要求在隐藏状态空间做统一方向的手术。它更像一个局部排序校准器。

同时，V2 也比 V1 更诚实：我们不再把“功能词绝对无损”表述成定理，也不再把 entropy jump 直接当作 hallucination detector。本质上，BRA-V2 只是把生成风险从“绝对视觉缺失”改写成“当前候选之间的相对证据不平衡”，并在 token 类型上引入保守的异质化处理。

## 6. Limitations

1. **BPE-aware damping 不是完整的 chunk aggregation。** 它只能缓解碎片误伤，不能从根本上解决多 token 实体的整体对齐问题。
2. **相对共振是局部排序信号，不是真值验证器。** 如果 Top-K 候选整体都缺乏视觉证据，BRA-V2 只能在坏候选里挑相对较好的那个。
3. **视频分支仍然是轻量启发式。** 即使加入公共运动抑制，它仍不足以处理长程复杂因果推理与强遮挡场景。
4. **跨模型泛化仍待验证。** 不同模型的 unembedding 结构和 tokenizer 规则不同，BRA-V2 的 token-aware 策略可能需要架构自适应。

## 7. Revised Claims for the Paper

V2 版本在写作上应遵守以下边界：

- 不宣称 SOTA，除非完整跑完统一协议下的主表。
- 不宣称“绝对语法无损”，改为“在 pilot 中观察到更稳定的句法保持”。
- 不宣称“视频动作显著提升”，改为“camera-motion-suppressed temporal branch 在视频动作 hallucination 上具有可检验潜力”。
- 不再使用旧标题中的 `Adaptive Orthogonal Subspace Projection`，统一收敛到 BRA。
